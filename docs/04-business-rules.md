# 04. Business Rules

## Главный идентификатор

Главный идентификатор автомобиля — нормализованный госномер.

Система должна нормализовать номер перед любой проверкой или записью.

## Количество АЗС

Система работает с 3 АЗС.

Каждая запись имеет `station_id`.

Но запрет повторной заправки действует по всем 3 АЗС.

## Повторная заправка

Обычный автомобиль может быть заправлен только один раз в календарный день.

Период:

```text
00:00 - 23:59 локальной даты
```

Проверка должна выполняться по:

```text
date + vehicle_id
```

А не по:

```text
date + station_id + vehicle_id
```

### Исключение для льготной очереди

Активная запись в льготной очереди разрешает дозаправку в тот же календарный день,
даже если автомобиль уже заправлялся по льготной записи.

`preferential_queue_entries.requested_liters` для активной записи хранит остаток
разрешённых литров. При фиксации заправки сервер вычитает фактически отпущенные
литры из этого остатка.

Если остаток больше нуля, льготная запись остаётся `ACTIVE`. Если остаток стал
нулём, запись получает статус `FUELED`.

Заправка по льготной записи запрещена, если запрошенный объём больше текущего
остатка литров.

## Заправка без записи

Если на текущую дату нет активной записи — заправка запрещена.

Исключение возможно только через ручное разрешение.

## Кто может дать ручное разрешение

Ручное разрешение может дать:

- мэр;
- управляющий АЗС.

Кассир и помощник мэра не могут давать ручное разрешение.

## Запись на дату

Запись доступна на любую дату.

По умолчанию интерфейс открывает завтрашнюю дату.

Один автомобиль не может иметь две активные записи на одну дату, даже на разных АЗС.

Проверка:

```text
date + vehicle_id
```

Для активных статусов:

- RESERVED
- ARRIVED
- APPROVED
- FUELING

## Лимиты

Лимит задаётся на дату и АЗС.

Есть:

- общий лимит машин;
- лимит машин по типу топлива;
- лимит литров на автомобиль;
- опциональный общий лимит литров по топливу.

Сумма лимитов по видам топлива не должна превышать общий лимит.

Для единой очереди по топливу дневная доступность считается по точным маркам:

- АИ-92;
- АИ-95;
- АИ-100;
- дизель;
- газ.

Категория `GASOLINE` не должна использоваться как взаимозаменяемость для всех бензинов при обзвоне и неявке.

## Предпочтение топлива

`fuel_reservations.fuel_type` хранит желаемую марку топлива.

`fuel_preference_mode` определяет совместимость:

- `EXACT` — подходит только выбранная марка;
- `ANY_GASOLINE` — подходят `AI_92`, `AI_95`, `AI_100`.

Существующие записи считаются `EXACT`, если пользователь явно не менял предпочтение.

Общий `queue_number` не меняется из-за отсутствия нужного топлива, смены предпочтения, фильтров или попадания в дневной лимит.

## Проверка допуска

Проверка допуска должна учитывать:

1. Госномер.
2. Нормализацию госномера.
3. АЗС.
4. Дату.
5. Наличие активной записи на дату.
6. Факт заправки по всем АЗС в этот день.
7. Лимиты.
8. Блокировки.
9. Ручные разрешения.
10. Статус синхронизации.

## Статусы допуска

### ALLOWED

Заправка разрешена.

### BLOCKED

Заправка запрещена.

### WARNING

Нужна проверка старшего смены.

## Офлайн-операции

Офлайн-операция не считается окончательно подтверждённой.

Она имеет статус:

- PENDING;
- SYNCING;
- SYNCED;
- FAILED;
- CONFLICT.

Если офлайн создана заправка, она должна быть перепроверена сервером при синхронизации.

Если офлайн создана отметка успешного звонка, сервер при синхронизации обязан заново проверить, что запись входила в актуальный список обзвона. При несовпадении операция получает `CONFLICT`.

## Отказ

Отказ должен сохраняться в `refusal_records`.

Причины:

- нет записи;
- уже заправлен;
- превышен лимит;
- приехал не в свою дату;
- запись отменена;
- запись просрочена;
- нет нужного топлива;
- решение старшего смены;
- другая причина.

## Удаление данных

Физически удалять историю нельзя.

Ошибочную запись нужно помечать статусом `ERROR`.

Все изменения — в `audit_logs`.

## Спецтранспорт

Для MVP спецтранспорт можно реализовать через `manual_overrides`.

## Главное правило

Все бизнес-правила должны быть продублированы на сервере.

Клиентская проверка нужна для удобства, но не является защитой.
