# 08. UI Guidelines

## Основной принцип

Приложение mobile-first.

Пользователь может работать с телефона, на улице, в спешке и при плохом интернете.

## Главные экраны

- Login
- Dashboard
- Проверка автомобиля
- Запись на дату
- Очередь сегодня
- Фиксация заправки
- Лимиты
- История
- Отчёты
- Синхронизация
- Пользователи
- Настройки

## Навигация

Для мобильного приложения использовать нижнюю навигацию.

Основные пункты:

- Проверка
- Запись
- Очередь
- Заправка
- Ещё

## Цветовые статусы

| Статус | Цвет |
|---|---|
| Разрешено | зелёный |
| Запрещено | красный |
| Нужна проверка | жёлтый |
| Ожидает | синий/нейтральный |
| Просрочено | серый |
| Конфликт | красный + предупреждение |
| Офлайн | жёлтый/серый |

## Проверка автомобиля

Экран должен иметь:

- крупное поле госномера;
- кнопку “Проверить”;
- быстрый выбор АЗС;
- результат крупным блоком;
- причину допуска или отказа.

## Запись на дату

По умолчанию открывается завтра.

Показывать:

- дату;
- АЗС;
- лимит;
- занято мест;
- свободно мест;
- лимит по топливу;
- кнопку создания записи.

## Форма записи

Поля:

- госномер;
- ФИО;
- телефон, необязательно;
- тип топлива;
- литры;
- комментарий.

После успешной записи показать номер записи.

## Фиксация заправки

Кассиру нужно минимум действий:

1. Ввести госномер.
2. Увидеть “Разрешено”.
3. Ввести фактические литры.
4. Нажать “Заправить”.

## Офлайн-баннер

Если нет интернета, сверху показывать:

```text
Вы офлайн. Действия сохраняются локально и будут синхронизированы позже.
```

- Button
- Input
- Form
- Select
- Dialog
- Sheet
- Card
- Badge
- Table только на больших экранах
- Toast/Sonner

- Tabs
- DropdownMenu

## Таблицы на мобильном

На телефоне не делать широкие таблицы.

Вместо таблицы использовать карточки.
