# 01. Product Requirements

## Назначение

Приложение нужно для контроля очереди, предварительной записи и отпуска топлива на 3 АЗС.

Система должна помогать:

- записывать автомобили на дату;
- контролировать лимиты;
- проверять автомобиль по госномеру;
- запрещать повторную заправку;
- фиксировать количество отпущенного топлива;
- давать администрации прозрачную статистику.

## Проблема

Из-за дефицита топлива на АЗС появляются очереди.

Без системы:

- люди пытаются заправляться повторно;
- сложно контролировать очередь;
- невозможно быстро проверить, кто был записан;
- нет единого учёта по 3 АЗС;
- появляются споры;
- нет отчётов по фактическому отпуску топлива.

## Пользователи

- оператор очереди;
- кассир;
- старший смены;
- администратор АЗС;
- администрация города;
- пользователь только для просмотра.

## Основные сценарии

### Создание лимита

Админ или старший смены создаёт лимит на дату:

- дата;
- АЗС;
- общий лимит автомобилей;
- лимиты по видам топлива;
- лимит литров на автомобиль;
- статус дня.

### Предварительная запись

Помощник мэра или управляющий АЗС выбирает дату, по умолчанию завтра, и записывает автомобиль:

- госномер;
- ФИО;
- телефон, необязательно;
- тип топлива;
- желаемые литры;
- АЗС;
- комментарий.

Система присваивает номер записи.

### Проверка автомобиля

Кассир, помощник мэра, управляющий АЗС или мэр вводит госномер.

Система показывает:

- разрешено;
- запрещено;
- нужна проверка;
- причину решения;
- последнюю заправку;
- активную запись.

### Фиксация заправки

Кассир после успешной проверки нажимает «Заправить» и фиксирует:

- госномер;
- тип топлива;
- фактическое количество литров;
- дату и время;
- АЗС;
- комментарий.

После этого автомобиль считается заправленным в этот календарный день по всем 3 АЗС.

### Ручное разрешение

Если есть спорная ситуация, старший смены или админ может дать ручное разрешение.

Все ручные разрешения записываются в журнал.

## MVP

MVP должен включать:

- авторизацию;
- роли;
- 3 АЗС;
- лимиты на дату;
- запись на любую дату;
- проверку госномера;
- запрет повторной заправки по всем АЗС;
- фиксацию заправки;
- отчёты;
- журнал действий;
- базовый офлайн-режим с синхронизацией.

## Не входит в MVP

- SMS;
- Telegram-бот;
- QR-коды;
- печать талонов;
- фото чеков;
- сканер госномеров;
- интеграция с кассой;
- публичная очередь для обычных людей.

## Критерии готовности MVP

MVP готов, если:

- админ создал 3 АЗС;
- админ или старший смены создал лимит на дату;
- оператор записал автомобиль;
- система не дала записать автомобиль второй раз на ту же дату;
- кассир проверил госномер;
- система показала корректный допуск;
- кассир зафиксировал заправку;
- повторная заправка в этот день запрещена по всем 3 АЗС;
- отчёт показывает количество машин и литров;
- действия попадают в audit log;
- офлайн-операции синхронизируются или попадают в конфликт.
