# 06. Roles and Permissions

## Роли

```text
operator
cashier
shift_supervisor
station_admin
city_admin
viewer
```

## Проверка прав

Права нужно проверять:

1. На уровне UI.
2. На уровне RPC.
3. Через RLS в Supabase.

UI-проверка нужна для удобства, но не является защитой.

## Доступ к АЗС

Через таблицу `user_stations`.

Примеры:

- оператор АЗС №1 видит только АЗС №1;
- кассир АЗС №2 видит только АЗС №2;
- station_admin видит свои АЗС;
- city_admin видит все 3 АЗС.

## Permission helpers

Нужны функции:

```text
canCreateReservation
canCreateFuelingRecord
canCreateDailyLimit
canCreateManualOverride
canViewAllStations
canResolveSyncConflict
```

## Важное правило

Нельзя прятать кнопку в UI и считать, что доступ защищён.

Любая критичная операция должна дополнительно проверять роль на сервере.
