# 06. Roles and Permissions

## Роли

```text
mayor
station_manager
cashier
mayor_assistant
```

## Назначение ролей

- `mayor` — мэр. Полный доступ ко всем 3 АЗС.
- `station_manager` — управляющий АЗС. Полный доступ только к назначенным АЗС.
- `cashier` — кассир АЗС. Может проверить, есть ли госномер в списке на заправку, и нажать «Заправить» после фактического отпуска топлива.
- `mayor_assistant` — помощник мэра. Может проверить госномер и добавить автомобиль в список.

## Проверка прав

Права нужно проверять:

1. На уровне UI.
2. На уровне RPC.
3. Через RLS в Supabase.

UI-проверка нужна для удобства, но не является защитой.

## Доступ к АЗС

Через таблицу `user_stations`, кроме городских ролей:

- `mayor` видит все 3 АЗС;
- `mayor_assistant` может проверять и добавлять записи по всем 3 АЗС;
- `station_manager` видит только назначенные АЗС;
- `cashier` видит только назначенные АЗС.

При создании записи в очередь система сохраняет автора в `fuel_queue_entries.operator_id`.
В интерфейсе очереди нужно показывать роль и ФИО автора записи.

## Permission helpers

Нужны функции:

```text
canCreateReservation
canCreateFuelingRecord
canCreateDailyLimit
canCreateManualOverride
canViewAllStations
canResolveSyncConflict
```

## Важное правило

Нельзя прятать кнопку в UI и считать, что доступ защищён.

Любая критичная операция должна дополнительно проверять роль на сервере.
